The Problem vs The Solution

Tradisional WAF

  • > Kaku, murni mengandalkan Pattern Matching (Regex).
  • > Sangat mudah di-bypass oleh serangan zero-day atau teknik manipulasi bahasa (obfuscation).
  • > Rawan False-Positive, sering memblokir pengguna asli secara keliru.
Sibyl-WAF

Intent-Based WAF

  • > Berbasis Niat (Intent-Based). Memahami tujuan payload melalui evaluasi LLM.
  • > Kebal terhadap teknik obfuscation bahasa tingkat tinggi dan mengenali anomali zero-day.
  • > Deterministik: Pemblokiran instan dan akurat berdasarkan skor Crime Coefficient (≥ 75).

Architecture Overview

The 2-Tier System

Lapis 1

Sibyl-Proxy

Reverse proxy ultra-ringan berbasis Golang di Home Server. Mencegat request privat (API/REST) dan merutekan traffic tanpa modifikasi pada target aplikasi.

Lapis 2

Sibyl-Brain

Mesin API Gateway analitik di Google Cloud Run. Membedah payload jaringan via Gemini API, mengevaluasi niat jahat, dan mengembalikan keputusan blokir instan.

Engineering Flex

FinOps & Skalabilitas

Arsitektur terdistribusi yang efisien. Pemisahan tugas proksi lokal dan otak kognitif cloud memastikan penggunaan token LLM yang optimal dan meminimalkan tagihan serverless.

CDN-Backed Polling

Distribusi Global Blacklist IP ditarik secara asinkron setiap 15 detik melalui jaringan CDN (Cloudflare). Tidak menambah beban latensi pada request pengguna normal.

Memory & Timeout Safety

Proteksi level rendah: Menggunakan io.LimitReader untuk mencegah pembengkakan heap (OOM) dari payload besar, dan strict context.WithTimeout (1.5s).
